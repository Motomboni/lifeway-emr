# User manual screenshots

PNG files here are generated by Playwright so **`Lifeway_EMR_User_Manual.md`** can show real UI.

## Generate images

1. Start the **backend** (e.g. `python manage.py runserver` on port 8000).
2. From the **`frontend/`** directory:

```bash
npx playwright test e2e/manual/capture-manual-screenshots.spec.ts --project=chromium
```

Or use the npm script:

```bash
npm run capture-manual-screenshots
```

Playwright may start the React dev server automatically if it is not already running.

## Required test users

Create at least **receptionist**, **doctor**, and **lab tech** accounts (see `frontend/e2e/billing/README.md`). Defaults assumed by the capture script:

| Role | Username (email) | Password |
|------|------------------|----------|
| Receptionist | `receptionist@clinic.com` | `Receptionist123!` |
| Doctor | `doctor@clinic.com` | `Doctor123!` |
| Lab tech | `labtech@clinic.com` | `LabTech123!` |

Override with environment variables: `MANUAL_RECEPTIONIST_EMAIL`, `MANUAL_RECEPTIONIST_PASSWORD`, `MANUAL_DOCTOR_EMAIL`, `MANUAL_DOCTOR_PASSWORD`, `MANUAL_LAB_EMAIL`, `MANUAL_LAB_PASSWORD`.

## Optional extra screenshots

If these are set, additional PNGs are written:

| Variables | Output files |
|-----------|----------------|
| `MANUAL_NURSE_EMAIL`, `MANUAL_NURSE_PASSWORD` | `11-nurse-dashboard.png` |
| `MANUAL_RADIOLOGY_EMAIL`, `MANUAL_RADIOLOGY_PASSWORD` | `12-radiology-orders.png` |
| `MANUAL_PHARMACIST_EMAIL`, `MANUAL_PHARMACIST_PASSWORD` | `13`–`15` pharmacist pages |
| `MANUAL_ADMIN_EMAIL`, `MANUAL_ADMIN_PASSWORD` | `16-admin-dashboard.png` |
| `MANUAL_PATIENT_EMAIL`, `MANUAL_PATIENT_PASSWORD` | `17-patient-portal-dashboard.png` |

## After regenerating

Commit updated PNGs if the manual should ship with current UI. The Markdown manual references these files by relative path (`manual-screenshots/…`).
